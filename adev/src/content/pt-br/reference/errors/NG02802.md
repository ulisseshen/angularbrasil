# Headers não transferidos pelo HttpTransferCache

Angular produz este aviso quando o código tenta acessar um header HTTP que não foi transferido do servidor para o cliente pelo `HttpTransferCache`.

Por padrão, `HttpTransferCache` não transfere nenhum header para evitar possíveis problemas de segurança ou transferência de dados desnecessários. Quando um header é acessado no cliente que não foi incluído, Angular registra este aviso para ajudá-lo a identificar headers ausentes.

## Corrigindo o erro

Para incluir headers específicos no cache de transferência, use a opção `includeHeaders`.

### No nível da requisição

Adicione o parâmetro `transferCache` à sua requisição HTTP:

```typescript
this.http.get('/api/data', {
  transferCache: {
    includeHeaders: ['cache-control', 'etag']
  }
});
```

### No nível da aplicação

Configure `includeHeaders` ao fornecer o cache de transferência:

```typescript
provideClientHydration(
  withHttpTransferCache({
    includeHeaders: ['cache-control', 'etag']
  })
);
```

Inclua apenas headers que sejam necessários para a lógica da sua aplicação. Tenha cuidado com headers sensíveis.

## Depurando o erro

A mensagem de aviso inclui o nome do header e a URL da requisição. Verifique seu código para onde este header está sendo acessado e determine se ele precisa ser incluído no cache de transferência.

Se o header não for necessário no cliente, considere refatorar seu código para evitar acessá-lo.
