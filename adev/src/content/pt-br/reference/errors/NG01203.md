# Value accessor ausente

Para todos os form controls customizados, você deve registrar um value accessor.

Aqui está um exemplo de como fornecer um:

```typescript
providers: [
  {
    provide: NG_VALUE_ACCESSOR,
    useExisting: forwardRef(() => MyInputField),
    multi: true,
  }
]
```

## Depurando o erro

Conforme descrito acima, seu control esperava ter um value accessor, mas estava faltando um. No entanto, há muitas razões diferentes pelas quais isso pode acontecer na prática. Aqui está uma lista de alguns problemas conhecidos que levam a este erro.

1. Se você **definiu** um form control customizado, você lembrou de fornecer um value accessor?
1. Você colocou `ngModel` em um elemento sem valor, ou em um **elemento inválido** (por exemplo, `<div [(ngModel)]="foo">`)?
1. Você está usando um form control customizado declarado dentro de um `NgModule`? Se sim, certifique-se de que você está **importando** o `NgModule`.
1. Você está usando `ngModel` com um form control customizado de terceiros? Verifique se esse control fornece um value accessor. Se não, use **`ngDefaultControl`** no elemento do control.
1. Você está **testando** um form control customizado? Certifique-se de configurar seu testbed para conhecer o control. Você pode fazer isso com `Testbed.configureTestingModule`.
1. Você está usando **Nx e Module Federation** com webpack? Seu `webpack.config.js` pode exigir [configuração extra](https://github.com/angular/angular/issues/43821#issuecomment-1054845431) para garantir que o pacote de forms seja compartilhado.
