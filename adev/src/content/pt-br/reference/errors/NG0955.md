# Expressão track resultou em chaves duplicadas para uma determinada coleção.

Uma expressão track especificada no loop `@for` foi avaliada como chaves duplicadas para uma determinada coleção, por exemplo:

```typescript
@Component({
    template: `@for (item of items; track item.value) {{{item.value}}}`,
})
class TestComponent {
    items = [{key: 1, value: 'a'}, {key: 2, value: 'b'}, {key: 3, value: 'a'}];
}
```

No exemplo fornecido, a expressão de rastreamento `item.key` encontrará duas chaves duplicadas `a` (no índice 0 e 2).

Chaves duplicadas são problemáticas do ponto de vista da correção: como o loop `@for` não pode identificar exclusivamente os itens, ele pode escolher nós DOM correspondentes a _outro_ item (com a mesma chave) ao realizar movimentações ou destruição de DOM.

Há também uma penalidade de desempenho associada a chaves duplicadas - internamente o Angular deve usar estruturas de dados mais sofisticadas e lentas ao repetir sobre coleções com chaves duplicadas.

## Corrigindo o erro

Altere a expressão de rastreamento para que ela identifique exclusivamente um item em uma coleção. No exemplo discutido, a expressão track correta usaria a propriedade `key` única (`item.key`):

```typescript
@Component({
    template: `@for (item of items; track item.key) {{{item.value}}}`,
})
class TestComponent {
    items = [{key: 1, value: 'a'}, {key: 2, value: 'b'}, {key: 3, value: 'a'}];
}
```
