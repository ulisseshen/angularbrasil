# Hydration com instância não suportada do Zone.js

Este aviso significa que a hydration foi habilitada para uma aplicação que foi configurada para usar uma versão não suportada do Zone.js: uma customizada ou "noop" (veja mais informações [aqui](api/core/BootstrapOptions#ngZone)).

Hydration depende de um sinal do Zone.js quando ele se torna estável dentro de uma aplicação, para que o Angular possa iniciar o processo de serialização no servidor ou a limpeza pós-hydration no cliente (para remover nós DOM que permaneceram não reclamados).

Fornecer uma implementação customizada ou "noop" do Zone.js pode levar a um timing diferente do evento "stable", acionando assim a serialização ou a limpeza muito cedo ou muito tarde. Esta ainda não é uma configuração totalmente suportada.

Se você usar uma implementação customizada do Zone.js, certifique-se de que o evento "onStable" seja emitido no momento certo e não resulte em comportamento incorreto da aplicação com hydration.

Mais informações sobre hydration podem ser encontradas no [guia de hydration](guide/hydration).
