# Comportamento do @defer quando HMR está habilitado

Hot Module Replacement (HMR) é uma técnica usada por servidores de desenvolvimento para evitar recarregar a página inteira quando apenas parte de uma aplicação é alterada.

Quando o HMR está habilitado no Angular, todas as dependências do bloco `@defer` são carregadas
de forma eager, ao invés de esperar por condições de trigger configuradas (tanto para triggers client-only quanto incremental hydration). Isso é necessário
para que o HMR funcione adequadamente, substituindo components em uma aplicação em tempo de execução
sem a necessidade de recarregar a página inteira. NOTA: a renderização real dos blocos defer
respeita as condições de trigger no modo HMR.

Se você quiser testar o comportamento do bloco `@defer` em modo de desenvolvimento e garantir que
as dependências necessárias sejam carregadas quando uma condição de trigger é atendida, você pode
desabilitar o modo HMR conforme descrito em [`este documento`](/tools/cli/build-system-migration#hot-module-replacement).
