# Hydration com instância não suportada de Zone.js

Este aviso significa que o hydration foi habilitado para uma aplicação que foi configurada para usar uma versão não suportada de Zone.js: seja uma customizada ou uma "noop" (veja mais informações [aqui](api/core/BootstrapOptions#ngZone)).

Hydration depende de um sinal do Zone.js quando ele se torna estável dentro de uma aplicação, para que o Angular possa iniciar o processo de serialização no servidor ou a limpeza pós-hydration no client (para remover nós DOM que permaneceram não reivindicados).

Fornecer uma implementação customizada ou "noop" de Zone.js pode levar a um timing diferente do evento "stable", disparando assim a serialização ou a limpeza muito cedo ou muito tarde. Esta ainda não é uma configuração totalmente suportada.

Se você usa uma implementação customizada de Zone.js, certifique-se de que o evento "onStable" é emitido no momento certo e não resulta em comportamento incorreto da aplicação com hydration.

Mais informações sobre hydration podem ser encontradas no [guia de hydration](guide/hydration).
