# No hydration info in server response

Este erro ocorre quando a hydration está ativada no client, mas a resposta do servidor
não contém informações especiais serializadas sobre a aplicação das quais a lógica de hydration
depende.

Isso pode acontecer quando a função `provideClientHydration()` está incluída na parte
client da configuração da aplicação, mas está faltando na parte server da configuração.

Em aplicações com a estrutura de projeto padrão (gerada pelo `ng new`),
a chamada `provideClientHydration()` é adicionada no array `providers` do
`AppModule` principal (que é importado no módulo do servidor) ou em um conjunto de
providers que são incluídos tanto nas configurações client quanto server.

Se você tem uma configuração personalizada onde as configurações client e server são independentes
e não compartilham um conjunto comum de providers, certifique-se de que o `provideClientHydration()`
também está incluído no conjunto de providers usado para fazer o bootstrap de uma aplicação no servidor.

Mais informações sobre hydration podem ser encontradas [neste guia](guide/hydration).

## Depurando o erro

Verifique se a chamada `provideClientHydration()` está incluída em um conjunto de providers
que é usado para fazer o bootstrap de uma aplicação no servidor.
