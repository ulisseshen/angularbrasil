# Dependência Circular em DI

<docs-video src="https://www.youtube.com/embed/CpLOm4o_FzM"/>

Uma dependência cíclica existe quando uma [dependência de um service](guide/di/hierarchical-dependency-injection) depende direta ou indiretamente do próprio service. Por exemplo, se `UserService` depende de `EmployeeService`, que também depende de `UserService`. O Angular terá que instanciar `EmployeeService` para criar `UserService`, que depende de `UserService`, ele mesmo.

## Depurando o erro

Use a pilha de chamadas para determinar onde a dependência cíclica existe.
Você poderá ver se alguma dependência filho depende do arquivo original [mapeando](guide/di/di-in-action) as dependências do component, module ou service, e identificando o loop que está causando o problema.

Quebre este loop \(ou círculo\) de dependência para resolver este erro. Isso geralmente significa remover ou refatorar as dependências para não serem dependentes uma da outra.
