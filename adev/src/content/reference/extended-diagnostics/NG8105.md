# Palavra-chave `let` ausente em uma expressão `*ngFor`

Este diagnóstico é emitido quando uma expressão usada em `*ngFor` está sem a palavra-chave `let`.

```typescript
import { Component } from '@angular/core';

@Component({
  // A palavra-chave `let` está ausente na expressão `*ngFor`.
  template: `<div *ngFor="item of items">{{ item }}</div>`,
})
class MyComponent {
  items = [1, 2, 3];
}
```

## Qual é o problema?

Um `let` ausente é indicativo de um erro de sintaxe na string `*ngFor`. Isso também significa que `item` não será corretamente trazido para o escopo e `{{ item }}` não será resolvido corretamente.

## O que devo fazer em vez disso?

Adicione a palavra-chave `let` ausente.

```typescript
import { Component } from '@angular/core';

@Component({
  // A palavra-chave `let` está agora presente na expressão `*ngFor`,
  // nenhuma mensagem de diagnóstico é emitida neste caso.
  template: `<div *ngFor="let item of items">{{ item }}</div>`,
})
class MyComponent {
  items = [1, 2, 3];
}
```

## Requisitos de configuração

[`strictTemplates`](tools/cli/template-typecheck#strict-mode) deve estar ativado para que qualquer diagnóstico estendido seja emitido.
`missingNgForOfLet` não possui requisitos adicionais além de `strictTemplates`.

## E se eu não puder evitar isso?

Este diagnóstico pode ser desativado editando o arquivo `tsconfig.json` do projeto:

<docs-code language="json">
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "missingNgForOfLet": "suppress"
      }
    }
  }
}
</docs-code>

Veja [configuração de diagnósticos estendidos](extended-diagnostics#configuration) para mais informações.
