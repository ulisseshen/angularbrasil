# `ngSkipHydration` deve ser um atributo estático

`ngSkipHydration` é um atributo especial que indica ao Angular que um component específico deve ser excluído da [hidratação](guide/hydration).
Este diagnóstico garante que este atributo `ngSkipHydration` seja definido estaticamente e o valor seja definido como `"true"` ou um valor vazio.

```typescript
import { Component } from '@angular/core';

@Component({
  template: `<user-viewer ngSkipHydration="hasUser" />`,
})
class MyComponent {
  hasUser = true;
}
```

## Qual é o problema?

Como um atributo especial implementado pelo Angular, `ngSkipHydration` precisa ser estaticamente analisável para que o Angular saiba em tempo de compilação se a hidratação é necessária ou não para um component.

## O que devo fazer em vez disso?

Ao usar o `ngSkipHydration`, certifique-se de que ele está definido como um atributo estático (ou seja, você não usa a sintaxe de binding de template do Angular).

```typescript
import { Component } from '@angular/core';

@Component({
  template: `
    <user-viewer ngSkipHydration />
    <user-viewer ngSkipHydration="true" />
  `,
})
class MyComponent {}
```

Se uma condicional for necessária, você pode envolver o component em um `*ngIf`.

<docs-code language="html">

import {Component} from '@angular/core';

@Component({
template: `

<div \*ngIf="hasUser; else noUser">
<user-viewer ngSkipHydration />
</div>

    <ng-template #noUser>
      <user-viewer />
    </ng-template>

`,
})
class MyComponent {}

</docs-code>

## Requisitos de configuração

[`strictTemplates`](tools/cli/template-typecheck#strict-mode) deve estar ativado para que qualquer diagnóstico estendido seja emitido.
`skipHydrationNotStatic` não possui requisitos adicionais além de `strictTemplates`.

## E se eu não puder evitar isso?

Este diagnóstico pode ser desativado editando o arquivo `tsconfig.json` do projeto:

<docs-code language="json">
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "skipHydrationNotStatic": "suppress"
      }
    }
  }
}
</docs-code>

Veja [configuração de diagnósticos estendidos](extended-diagnostics#configuration) para mais informações.
