# Funções devem ser invocadas em event bindings.

Este diagnóstico detecta funções não invocadas em event bindings.

```typescript
import { Component, signal, Signal } from '@angular/core';

@Component({
  template: `<button (click)="(onClick)">Click me</button>`,
})
class MyComponent {
  onClick() {
    console.log('clicked');
  }
}
```

## O que há de errado com isso?

Funções em event bindings devem ser invocadas quando o evento é disparado.
Se a função não for invocada, ela não será executada quando o evento for disparado.

## O que devo fazer em vez disso?

Certifique-se de invocar a função quando você a usa em um event binding para executar a função quando o evento for disparado.

```typescript
import { Component } from '@angular/core';

@Component({
  template: `<button (click)="onClick()">Click me</button>`,
})
class MyComponent {
  onClick() {
    console.log('clicked');
  }
}
```

## Requisitos de configuração

[`strictTemplates`](tools/cli/template-typecheck#strict-mode) deve estar ativado para que qualquer diagnóstico estendido seja emitido.
`uninvokedFunctionInEventBinding` não tem requisitos adicionais além de `strictTemplates`.

## E se eu não puder evitar isso?

Este diagnóstico pode ser desativado editando o arquivo `tsconfig.json` do projeto:

<docs-code language="json">
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "uninvokedFunctionInEventBinding": "suppress"
      }
    }
  }
}
</docs-code>

Veja [configuração de diagnóstico estendido](extended-diagnostics#configuration) para mais informações.
