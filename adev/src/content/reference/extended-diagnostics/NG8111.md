# Funções devem ser invocadas em bindings de eventos.

Este diagnóstico detecta funções não invocadas em bindings de eventos.

```typescript
import { Component, signal, Signal } from '@angular/core';

@Component({
  template: `<button (click)="(onClick)">Click me</button>`,
})
class MyComponent {
  onClick() {
    console.log('clicked');
  }
}
```

## Qual é o problema?

Funções em bindings de eventos devem ser invocadas quando o evento é acionado.
Se a função não for invocada, ela não será executada quando o evento for acionado.

## O que devo fazer em vez disso?

Certifique-se de invocar a função quando você a usa em um binding de evento para executar a função quando o evento for acionado.

```typescript
import { Component } from '@angular/core';

@Component({
  template: `<button (click)="onClick()">Click me</button>`,
})
class MyComponent {
  onClick() {
    console.log('clicked');
  }
}
```

## Requisitos de configuração

[`strictTemplates`](tools/cli/template-typecheck#strict-mode) deve estar ativado para que qualquer diagnóstico estendido seja emitido.
`uninvokedFunctionInEventBinding` não possui requisitos adicionais além de `strictTemplates`.

## E se eu não puder evitar isso?

Este diagnóstico pode ser desativado editando o arquivo `tsconfig.json` do projeto:

<docs-code language="json">
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "uninvokedFunctionInEventBinding": "suppress"
      }
    }
  }
}
</docs-code>

Veja [configuração de diagnósticos estendidos](extended-diagnostics#configuration) para mais informações.
