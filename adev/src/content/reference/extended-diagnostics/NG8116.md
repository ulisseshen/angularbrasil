# Structural directive ausente

Este diagnóstico garante que um component standalone que usa structural directives customizadas (por exemplo, `*select` ou `*featureFlag`) em um template tenha os imports necessários para essas directives.

```typescript
import { Component } from '@angular/core';

@Component({
  // Template uses `*select`, but no corresponding directive imported.
  imports: [],
  template: `<p *select="let data from source">{{ data }}</p>`,
})
class MyComponent {}
```

## Qual é o problema?

Usar uma structural directive sem importá-la falhará em runtime, pois o Angular tentará fazer binding com uma property `select` do elemento HTML, que não existe.

## O que devo fazer em vez disso?

Certifique-se de que a structural directive correspondente está importada no component:

```typescript
import { Component } from '@angular/core';
import { SelectDirective } from 'my-directives';

@Component({
  // Add `SelectDirective` to the `imports` array to make it available in the template.
  imports: [SelectDirective],
  template: `<p *select="let data from source">{{ data }}</p>`,
})
class MyComponent {}

```

## Requisitos de configuração

[`strictTemplates`](tools/cli/template-typecheck#strict-mode) deve estar habilitado para que qualquer diagnóstico estendido seja emitido.
`missingStructuralDirective` não tem requisitos adicionais além de `strictTemplates`.

## E se eu não puder evitar isso?

Este diagnóstico pode ser desabilitado editando o arquivo `tsconfig.json` do projeto:

<docs-code language="json">
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "missingStructuralDirective": "suppress"
      }
    }
  }
}
</docs-code>

Veja [configuração de diagnósticos estendidos](extended-diagnostics#configuration) para mais informações.
