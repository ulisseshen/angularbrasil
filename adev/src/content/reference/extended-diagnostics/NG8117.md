# Functions devem ser invocadas em interpolação de texto.

Este diagnóstico detecta functions não invocadas em interpolação de texto.

```typescript
import { Component } from '@angular/core';

@Component({
  template: `{{ getValue }}`,
})
class MyComponent {
  getValue() {
    return 'value';
  }
}
```

## O que há de errado com isso?

Functions em interpolação de texto devem ser invocadas para retornar um valor.
Se a function não for invocada, ela não retornará nenhum valor e a interpolação não funcionará como esperado.

## O que devo fazer em vez disso?

Certifique-se de invocar a function quando você a usar em interpolação de texto para retornar o valor.

```typescript
import { Component } from '@angular/core';

@Component({
  template: `{{ getValue() }}`,
})
class MyComponent {
  getValue() {
    return 'value';
  }
}
```

## Requisitos de configuração

[`strictTemplates`](/tools/cli/template-typecheck#strict-mode) deve estar habilitado para que qualquer diagnóstico estendido seja emitido.
`uninvokedFunctionInTextInterpolation` não possui requisitos adicionais além de `strictTemplates`.

## E se eu não puder evitar isso?

Este diagnóstico pode ser desabilitado editando o arquivo `tsconfig.json` do projeto:

<docs-code language="json">
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "uninvokedFunctionInTextInterpolation": "suppress"
      }
    }
  }
}
</docs-code>

Veja [configuração de diagnósticos estendidos](/extended-diagnostics#configuration) para mais informações.
