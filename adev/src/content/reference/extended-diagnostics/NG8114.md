# Nullish Coalescing sem Parênteses

Este diagnóstico detecta casos onde o operador nullish coalescing (`??`) é misturado com os operadores logical or (`||`) ou logical and (`&&`) sem parênteses para desambiguar a precedência.

```typescript
import { Component, signal, Signal } from '@angular/core';

@Component({
  template: `
    <button [disabled]="hasPermission() && (task()?.disabled ?? true)">
      Run
    </button>
  `,
})
class MyComponent {
  hasPermission = input(false);
  task = input<Task | undefined>(undefined);
}
```

## O que há de errado nisso?

Sem parênteses para desambiguação, é difícil entender se o `??` ou `||`/`&&` é avaliado primeiro. Isso é considerado um erro em TypeScript e JavaScript, mas historicamente foi permitido em templates Angular.

## O que devo fazer em vez disso?

Sempre use parênteses para desambiguar nessas situações. Se você não tem certeza de qual era a intenção original do código mas quer manter o comportamento igual, coloque os parênteses ao redor do operador `??`.

```typescript
import { Component, signal, Signal } from '@angular/core';

@Component({
  template: `
    <button [disabled]="hasPermission() && (task()?.disabled ?? true)">
      Run
    </button>
  `,
})
class MyComponent {
  hasPermission = input(false);
  task = input<Task | undefined>(undefined);
}
```
