# Directive de controle de fluxo ausente

Este diagnóstico garante que um component standalone que usa directives de controle de fluxo conhecidas
(como `*ngIf`, `*ngFor` ou `*ngSwitch`) em um template, também importe essas directives individualmente
ou importando o `CommonModule`.

<docs-code language="typescript">

import {Component} from '@angular/core';

@Component({
// Template usa `*ngIf`, mas nenhuma directive correspondente importada.
imports: [],
template: `<div *ngIf="visible">Hi</div>`,
})
class MyComponent {}

</docs-code>

## Qual é o problema?

Usar uma directive de controle de fluxo sem importá-la falhará em runtime, pois o Angular tenta fazer binding com uma propriedade `ngIf` do elemento HTML, que não existe.

## O que devo fazer em vez disso?

Certifique-se de que uma directive de controle de fluxo correspondente seja importada.

Uma directive pode ser importada individualmente:

<docs-code language="typescript">

import {Component} from '@angular/core';
import {NgIf} from '@angular/common';

@Component({
imports: [NgIf],
template: `<div *ngIf="visible">Hi</div>`,
})
class MyComponent {}

</docs-code>

ou você pode importar todo o `CommonModule`, que contém todas as directives de controle de fluxo:

<docs-code language="typescript">

import {Component} from '@angular/core';
import {CommonModule} from '@angular/common';

@Component({
imports: [CommonModule],
template: `<div *ngIf="visible">Hi</div>`,
})
class MyComponent {}

</docs-code>

## Requisitos de configuração

[`strictTemplates`](tools/cli/template-typecheck#strict-mode) deve estar habilitado para que qualquer diagnóstico estendido seja emitido.
`missingControlFlowDirective` não tem requisitos adicionais além de `strictTemplates`.

## E se eu não puder evitar isso?

Este diagnóstico pode ser desabilitado editando o arquivo `tsconfig.json` do projeto:

<docs-code language="json">
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "missingControlFlowDirective": "suppress"
      }
    }
  }
}
</docs-code>

Veja [configuração de diagnósticos estendidos](extended-diagnostics#configuration) para mais informações.
