# Função Track Não Invocada

Este diagnóstico detecta quando uma função track não é invocada em blocos `@for`.

```typescript
import { Component } from '@angular/core';

@Component({
  template: `@for (item of items; track trackByName) {}`,
})
class Items {
  protected trackByName(item) {
    return item.name;
  }
}
```

## Qual é o problema?

Blocos `@for` precisam identificar unicamente os itens no iterável para executar corretamente as atualizações do DOM quando os itens no iterável são reordenados, novos itens são adicionados ou itens existentes são removidos.
Quando você não invoca a função, o algoritmo de reconciliação usará a referência da função ao invés do valor retornado para comparar os itens.

## O que devo fazer em vez disso?

Certifique-se de invocar a função track quando você usá-la em um bloco `@for` para executar a função e permitir que o loop identifique unicamente os itens.

```typescript
import { Component } from '@angular/core';

@Component({
  template: `@for (item of items; track trackByName(item)) {}`,
})
class Items {
  protected trackByName(item) {
    return item.name;
  }
}
```

## Requisitos de configuração

[`strictTemplates`](tools/cli/template-typecheck#strict-mode) deve estar ativado para que qualquer extended diagnostic seja emitido.
`uninvokedTrackFunction` não possui requisitos adicionais além de `strictTemplates`.

## E se eu não puder evitar isso?

Este diagnóstico pode ser desativado editando o arquivo `tsconfig.json` do projeto:

<docs-code language="json">
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "uninvokedTrackFunction": "suppress"
      }
    }
  }
}
</docs-code>

Veja [configuração de extended diagnostic](extended-diagnostics#configuration) para mais informações.
