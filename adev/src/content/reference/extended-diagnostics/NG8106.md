# Sufixo não suportado

Este diagnóstico detecta quando os sufixos `.px`, `.%` e `.em` são usados com um
binding de atributo.

<docs-code language="html">

<img [attr.width.px]="5">

</docs-code>

## Qual é o problema?

Esses sufixos estão disponíveis apenas para bindings de estilo. Eles não têm nenhum significado ao fazer binding com um atributo.

## O que devo fazer em vez disso?

Em vez de usar os sufixos `.px`, `.%` ou `.em` que são suportados apenas em bindings de estilo,
mova isso para a atribuição de valor do binding.

<docs-code language="html">

<img [attr.width]="'5px'">

</docs-code>

## Requisitos de configuração

[`strictTemplates`](tools/cli/template-typecheck#strict-mode) deve estar ativado para que qualquer diagnóstico estendido seja emitido.
`suffixNotSupported` não possui requisitos adicionais além de `strictTemplates`.

## E se eu não puder evitar isso?

Este diagnóstico pode ser desativado editando o arquivo `tsconfig.json` do projeto:

<docs-code language="json">
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "suffixNotSupported": "suppress"
      }
    }
  }
}
</docs-code>

Veja [configuração de diagnósticos estendidos](extended-diagnostics#configuration) para mais informações.
