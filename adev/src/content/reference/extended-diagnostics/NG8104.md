# Atributo de texto não é binding

Este diagnóstico garante que atributos que possuem o prefixo de binding "especial" do Angular (`attr.`, `style.` e
`class.`) sejam interpretados como bindings.

<docs-code language="html">

<div attr.id="my-id"></div>

</docs-code>

## Qual é o problema?

Neste exemplo, `attr.id` é interpretado como um atributo regular e aparecerá
como está no HTML final (`<div attr.id="my-id"></div>`). Provavelmente não é a intenção do desenvolvedor.
Em vez disso, a intenção provável é definir o atributo `id` (`<div id="my-id"></div>`).

## O que devo fazer em vez disso?

Ao fazer binding com `attr.`, `class.` ou `style.`, certifique-se de usar a sintaxe de binding de template do Angular (`[]`).

<docs-code language="html">

<div [attr.id]="my-id"></div>
<div [style.color]="red"></div>
<div [class.large]="true"></div>

</docs-code>

## Requisitos de configuração

[`strictTemplates`](tools/cli/template-typecheck#strict-mode) deve estar ativado para que qualquer diagnóstico estendido seja emitido.
`textAttributeNotBinding` não possui requisitos adicionais além de `strictTemplates`.

## E se eu não puder evitar isso?

Este diagnóstico pode ser desativado editando o arquivo `tsconfig.json` do projeto:

<docs-code language="json">

{
"angularCompilerOptions": {
"extendedDiagnostics": {
"checks": {
"textAttributeNotBinding": "suppress"
}
}
}
}

</docs-code>

Veja [configuração de diagnósticos estendidos](extended-diagnostics#configuration) para mais informações.
