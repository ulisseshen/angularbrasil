# Invalid Banana-in-Box

Este diagnóstico detecta uma sintaxe "banana-in-box" invertida para [bindings bidirecionais](guide/templates/two-way-binding).

<docs-code language="html">

<user-viewer ([user])="loggedInUser" />

</docs-code>

## O que há de errado com isso?

Da forma como está, `([var])` é na verdade um [event binding](guide/templates/event-listeners) com o nome `[var]`.
O autor provavelmente pretendia isso como um binding bidirecional para uma variável chamada `var`, mas, como escrito, este binding não tem efeito.

## O que devo fazer em vez disso?

Corrija o erro de digitação.
Como o nome sugere, a banana `(` deve ir _dentro_ da caixa `[]`.
Neste caso:

<docs-code language="html">

<user-viewer [(user)]="loggedInUser" />

</docs-code>

## Requisitos de configuração

[`strictTemplates`](tools/cli/template-typecheck#strict-mode) deve estar habilitado para qualquer diagnóstico estendido emitir.
`invalidBananaInBox` não tem requisitos adicionais além de `strictTemplates`.

## E se eu não puder evitar isso?

Este diagnóstico pode ser desabilitado editando o arquivo `tsconfig.json` do projeto:

<docs-code language="json">
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "invalidBananaInBox": "suppress"
      }
    }
  }
}
</docs-code>

Veja [configuração de diagnósticos estendidos](extended-diagnostics#configuration) para mais informações.
