# Signals devem ser invocados em interpolações de template.

Este diagnóstico detecta signals não invocados em interpolações de template.

```typescript
import { Component, signal, Signal } from '@angular/core';

@Component({
  template: `<div>{{ mySignal }}</div>`,
})
class MyComponent {
  mySignal: Signal<number> = signal(0);
}
```

## Qual é o problema?

Angular Signals são funções de zero argumentos (`() => T`). Quando executadas, elas retornam o valor atual do signal.
Isso significa que elas devem ser invocadas quando usadas em interpolações de template para renderizar seu valor.

## O que devo fazer em vez disso?

Certifique-se de invocar o signal quando você o usa dentro de uma interpolação de template para renderizar seu valor.

```typescript
import { Component, signal, Signal } from '@angular/core';

@Component({
  template: `<div>{{ mySignal() }}</div>`,
})
class MyComponent {
  mySignal: Signal<number> = signal(0);
}
```

## Requisitos de configuração

[`strictTemplates`](tools/cli/template-typecheck#strict-mode) deve estar ativado para que qualquer diagnóstico estendido seja emitido.
`interpolatedSignalNotInvoked` não possui requisitos adicionais além de `strictTemplates`.

## E se eu não puder evitar isso?

Este diagnóstico pode ser desativado editando o arquivo `tsconfig.json` do projeto:

<docs-code language="json">
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "interpolatedSignalNotInvoked": "suppress"
      }
    }
  }
}
</docs-code>

Veja [configuração de diagnósticos estendidos](extended-diagnostics#configuration) para mais informações.
