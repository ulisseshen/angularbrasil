## Suporte a browsers

O Angular usa a ["amplamente disponível" Baseline](https://web.dev/baseline) para definir o suporte a browsers. Para cada versão major, o Angular suporta browsers incluídos na Baseline de uma data escolhida próxima à data de lançamento dessa major.

A Baseline "amplamente disponível" inclui browsers lançados há menos de 30 meses (2,5 anos) da data escolhida dentro do conjunto central de browsers da Baseline (Chrome, Edge, Firefox, Safari) e visa suportar aproximadamente 95% dos usuários da web.

| Angular | Data Baseline | Conjunto de Browsers        |
| ------- | ------------- | --------------------------- |
| v21     | 2025-10-20    | [Conjunto de Browsers][browsers-v21] |
| v20     | 2025-04-30    | [Conjunto de Browsers][browsers-v20] |

[browsers-v21]: https://web-platform-dx.github.io/web-features/supported-browsers/?widelyAvailableOnDate=2025-10-20&includeDownstream=false
[browsers-v20]: https://web-platform-dx.github.io/web-features/supported-browsers/?widelyAvailableOnDate=2025-04-30&includeDownstream=false

Versões do Angular anteriores à v20 suportam as seguintes versões específicas de browsers:

| Browser | Versões suportadas                      |
| :------ | :-------------------------------------- |
| Chrome  | 2 versões mais recentes                 |
| Firefox | versão mais recente e extended support release \(ESR\) |
| Edge    | 2 versões major mais recentes           |
| Safari  | 2 versões major mais recentes           |
| iOS     | 2 versões major mais recentes           |
| Android | 2 versões major mais recentes           |

## Polyfills

O Angular é construído sobre os padrões mais recentes da plataforma web.
Ter como alvo uma gama tão ampla de browsers é desafiador porque eles não suportam todos os recursos de browsers modernos.
Você compensa isso carregando scripts polyfill ("polyfills") para os browsers que você deve suportar.
Veja instruções sobre como incluir polyfills em seu projeto abaixo.

IMPORTANTE: Os polyfills sugeridos são aqueles que executam aplicações Angular completas.
Você pode precisar de polyfills adicionais para suportar funcionalidades não cobertas por esta lista.

ÚTIL: Polyfills não podem magicamente transformar um browser antigo e lento em um moderno e rápido.

## Habilitando polyfills com projetos CLI

O [Angular CLI](tools/cli) fornece suporte para polyfills.
Se você não está usando o CLI para criar seus projetos, veja [Instruções de polyfill para usuários que não usam CLI](#polyfills-for-non-cli-users).

A opção `polyfills` dos [builders browser e test](tools/cli/cli-builder) pode ser um caminho completo para um arquivo \(Exemplo: `src/polyfills.ts`\) ou,
relativo ao workspace atual ou especificador de módulo \(Exemplo: `zone.js`\).

Se você criar um arquivo TypeScript, certifique-se de incluí-lo na propriedade `files` do seu arquivo `tsconfig`.

<docs-code language="json">
{
  "extends": "./tsconfig.json",
  "compilerOptions": {
    ...
  },
  "files": [
    "src/main.ts",
    "src/polyfills.ts"
  ]
  ...
}
</docs-code>

## Polyfills para usuários que não usam CLI

Se você não está usando o CLI, adicione seus scripts de polyfill diretamente à página web host \(`index.html`\).

Por exemplo:

<docs-code header="src/index.html" language="html">
<!-- pre-zone polyfills -->
<script src="node_modules/core-js/client/shim.min.js"></script>
<script>
  /**
   * você pode configurar algumas flags de zone que podem desabilitar a interceptação de zone para algumas
   * atividades assíncronas para melhorar a performance de inicialização - use essas opções apenas
   * se você sabe o que está fazendo, pois pode resultar em bugs difíceis de rastrear.
   */
  // __Zone_disable_requestAnimationFrame = true; // desabilitar patch requestAnimationFrame
  // __Zone_disable_on_property = true; // desabilitar patch onProperty como onclick
  // __zone_symbol__UNPATCHED_EVENTS = ['scroll', 'mousemove']; // desabilitar patch de eventNames especificados
  /*
   * nas ferramentas de desenvolvedor do Edge, o addEventListener também será envolvido por zone.js
   * com a seguinte flag, ele irá ignorar o patch `zone.js` para Edge.
   */
  // __Zone_enable_cross_context_check = true;
</script>
<!-- zone.js requerido pelo Angular -->
<script src="node_modules/zone.js/bundles/zone.umd.js"></script>
<!-- application polyfills -->
</docs-code>
